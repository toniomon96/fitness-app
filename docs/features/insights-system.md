# Insights System

Omnexus surfaces three distinct AI-powered insight layers: a training analysis report, a session adaptation engine, and a pre-workout briefing. Together these form a continuous feedback loop around every workout.

---

## 1. Training Insights (`/api/insights`)

A periodic training analysis report generated by Claude.

### What It Does
- Analyzes the user's last 4 weeks of workout history
- Identifies volume trends, frequency patterns, and recovery signals
- Produces personalized recommendations for the next training phase
- Surfaced on `src/pages/InsightsPage.tsx`

### Request
```http
POST /api/insights
Authorization: Bearer <supabase-access-token>
Content-Type: application/json

{
  "workoutSummary": "<string — pre-built in insightsService.ts>",
  "profile": { "goal": "hypertrophy", "trainingAge": "intermediate" }
}
```

### Response
```json
{
  "insights": "Your pushing volume is strong but pulling has dropped 20% over the last three weeks...",
  "recommendations": ["...", "..."]
}
```

### Client-Side
- `src/services/insightsService.ts` — builds the `workoutSummary` string from `WorkoutSession[]`
- `src/components/insights/AdaptationCard.tsx` — displays the AI insight narrative

---

## 2. Adaptation Engine (`/api/adapt`)

Per-exercise session adjustments generated by Claude after every completed workout.

### What It Does
- Receives the set-level log for the completed session
- Analyzes RPE trends per exercise over the last 2–3 sessions
- Outputs exercise-by-exercise coaching notes for the *next* session (e.g., "Add 2.5 kg to bench — RPE was 7 on the last two sets")
- Result stored in `localStorage` under `omnexus_last_adaptation`

### Trigger
Called from `WorkoutCompleteModal` when it opens after a session completes.

### Request
```http
POST /api/adapt
Authorization: Bearer <supabase-access-token>   (optional — userId in body)
Content-Type: application/json

{
  "userId": "...",
  "sessionLog": [ ... ],
  "recentHistory": [ ... ]
}
```

### Response
```json
{
  "adaptations": [
    { "exerciseId": "barbell-bench-press", "note": "Add 2.5 kg next session", "direction": "increase" },
    ...
  ]
}
```

### UI
- Adaptation results shown in `WorkoutCompleteModal` and in the adaptation banner in `ActiveWorkoutPage`
- `src/components/insights/AdaptationCard.tsx` — displays adaptation recommendations

---

## 3. Pre-Workout Briefing (`/api/briefing`)

A short coach-authored briefing generated before a workout begins.

### What It Does
- Reviews the workout scheduled for today against recent session data
- Highlights form reminders and key volume targets
- Flags if the user is due for a deload based on accumulated fatigue signals
- Surfaced on `src/pages/PreWorkoutBriefingPage.tsx`

### Request
```http
POST /api/briefing
Authorization: Bearer <supabase-access-token>
Content-Type: application/json

{
  "userId": "...",
  "scheduledWorkout": { ... },
  "recentHistory": [ ... ]
}
```

---

## 4. Peer Insights (`/api/peer-insights`)

Anonymous aggregate comparisons against users with similar training profiles.

### What It Does
- Queries anonymized volume and frequency data for users in the same goal/experience bucket
- Returns benchmark comparisons ("Athletes like you average 3.8 sessions/week")
- Never exposes individual user data

### UI
- `src/components/insights/PeerInsightsCard.tsx`

---

## Insight Loop

The full insight cycle per user:
```
Pre-Workout Briefing  →  Active Workout  →  Adaptation Engine  →  WorkoutCompleteModal
                                                   ↓
                              localStorage[omnexus_last_adaptation]
                                                   ↓
                              Surfaces in next Pre-Workout Briefing
                                                   ↓
                              Training Insights (4-week aggregate)
```
